## 1. 데이터 삽입 — INSERT

```sql
-- 기본
INSERT INTO [테이블명] ([컬럼1], [컬럼2], [컬럼3])
VALUES ([값1], [값2], [값3]);

-- 컬럼 순서 변경 가능
INSERT INTO [테이블명] ([컬럼2], [컬럼1], [컬럼3])
VALUES ([값2], [값1], [값3]);

-- 컬럼 이름 생략 (테이블 컬럼 순서대로 입력)
INSERT INTO [테이블명]
VALUES ([값1], [값2], [값3]);

-- 일부 컬럼만 입력 (나머지는 NULL 또는 DEFAULT)
INSERT INTO [테이블명] ([컬럼1], [컬럼2])
VALUES ([값1], [값2]);
```

> 더미 데이터 생성 도구: [Mockaroo](https://www.mockaroo.com), [Generatedata](https://generatedata.com)

### INSERT ... SELECT

다른 테이블에서 조회한 결과를 그대로 INSERT:

```sql
INSERT INTO orders (email, product, qty)
SELECT email, 'Keyboard', 1
FROM source_users
WHERE email = 'alice@example.com';
```

## 2. 데이터 조회 — SELECT

### 기본 조회

```sql
SELECT * FROM [테이블명];                           -- 전체 컬럼
SELECT [컬럼1], [컬럼2] FROM [테이블명];            -- 특정 컬럼만
```

### AS — 컬럼 별칭

실제 테이블 구조는 바뀌지 않고 조회 결과에서만 이름이 바뀜.

```sql
SELECT [컬럼명] AS [별칭] FROM [테이블명];
SELECT 10*5 AS result;
```

- 별칭은 선택 사항
- 조회 결과에서만 표시되고 DB 구조에는 영향 없음 (일시적)
- 일반 컬럼은 AS 생략 시 원래 컬럼명 그대로 표시됨 (`SELECT name` → `name`)
- 계산식/함수는 AS 생략 시 의미없는 이름으로 표시됨
  - 계산식 : `SELECT 10*5` → `?column?`
  - 함수 : `SELECT AVG(price)` → `avg`
  - AS 사용 : `SELECT 10*5 AS result` → `result`

**쓰는 이유**
- 프론트엔드가 원하는 JSON 키 이름으로 맞춰줄 때 (`user_id` → `userId`)
- 계산식이나 함수 결과에 의미 있는 이름을 붙일 때
- JOIN 시 같은 이름의 컬럼이 충돌할 때 구분하기 위해

실무 활용:
```sql
-- 프론트엔드가 원하는 JSON 키 이름으로 맞춰줄 때
SELECT user_id AS "userId", created_at AS "createdAt" FROM users;

-- JOIN 시 컬럼 이름 충돌 방지 (뒤에서 등장)
```

### WHERE — 조건 필터링

조회할 데이터의 조건을 지정. 조건에 맞는 행만 반환.

1) 단순 조건
```sql
SELECT * FROM users WHERE status = 'active';

-- [결과]
-- ('alice', 'active'), ('bob', 'active')
```

2) OR — 조건 중 하나라도 맞으면
```sql
SELECT * FROM users WHERE status = 'active' OR status = 'pending';

-- [결과]
-- ('alice', 'active'), ('carol', 'pending')
```

3) AND — 조건 둘 다 맞아야
```sql
SELECT * FROM users WHERE status = 'active' AND age >= 20;

-- [결과]
-- ('alice', 'active', 25)
```

4) 괄호로 우선순위 묶기
```sql
SELECT * FROM users WHERE (status = 'active' OR status = 'pending') AND age >= 20;

-- [결과]
-- ('alice', 'active', 25), ('carol', 'pending', 22)
```

### LIKE — 패턴 검색

문자열에서 특정 패턴을 찾는 검색.

| 와일드카드 | 의미 | 상세 |
|-----------|------|------|
| `%` | 0개 이상의 임의 문자 | **위치**만 맞으면 됨. 개수 자유 |
| `_` | 정확히 1개의 임의 문자 | **위치 + 개수** 둘 다 정확히 맞아야 함 |

1) `%` — **위치**가 핵심 (개수 상관없음)
```sql
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE 'a%';  -- a로 시작
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE '%a';  -- a로 끝남
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE '%a%'; -- a가 포함
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE '%CA'; -- CA로 끝나는 값

-- [결과] LIKE 'a%'
-- ('alice'), ('a'), ('abcdefg')
-- ('balice') ❌ → % 앞 a가 없음

-- [결과] LIKE '%CA' → % 위치가 앞이니까 앞에 뭐가 몇 개 오든 상관없음
-- ('San Francisco, CA'), ('Los Angeles, CA'), ('San Diego, CA')
```

2) `_` — **위치 + 개수** 둘 다 정확히 맞아야 함
```sql
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE 'a_';       -- a 뒤에 딱 1글자
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE '_____@%';  -- @ 앞에 정확히 5글자

-- [결과] LIKE 'a_'
-- ('ab'), ('a1')
-- ('alice') ❌ → _ 1개인데 뒤에 4글자 / ('a') ❌ → _ 1개인데 뒤에 0글자

-- [결과] LIKE '_____@%' → _ 5개니까 @ 앞 글자 수가 정확히 5개여야 함, @ 뒤는 % 라서 상관없음
-- ('alice@gmail.com'), ('boost@naver.com'), ('hello@kakao.com')
-- ('al@gmail.com') ❌ → @ 앞이 2글자 / ('123456@gmail.com') ❌ → @ 앞이 6글자
```

### BETWEEN — 범위 검색

특정 범위 안에 있는 데이터를 찾는 검색.

| 특징 | 설명 |
|------|------|
| 끝값 | A, B 자기 자신 포함 (`>=`, `<=`) |
| 타입 | 숫자, 날짜, 문자열 |
| NULL | NULL 값은 범위에서 제외됨 |

1) 숫자 범위
```sql
SELECT * FROM [테이블명] WHERE age BETWEEN 20 AND 30;

-- [결과] 20 이상 30 이하 (양 끝값 포함)
-- ('alice', 25), ('bob', 20), ('carol', 30)
-- ('dave', 19) ❌ → 20 미만 / ('eve', 31) ❌ → 30 초과
```

2) 날짜 범위
```sql
SELECT * FROM [테이블명] WHERE birthday BETWEEN '1975-01-01' AND '1980-12-31';

-- [결과] 1975년 1월 1일 ~ 1980년 12월 31일 사이
-- ('1975-01-01'), ('1978-06-15'), ('1980-12-31')
-- ('1974-12-31') ❌ → 범위 이전 / ('1981-01-01') ❌ → 범위 이후
```

3) 문자열 범위
```sql
SELECT * FROM [테이블명] WHERE name BETWEEN 'A' AND 'M';

-- [결과] 알파벳 순서로 A ~ M 사이
-- ('Alice'), ('Bob'), ('Dave'), ('Mike')
-- ('Nathan') ❌ → N은 M 이후 / ('Zoe') ❌ → Z는 M 이후
```

> `>= A AND <= B` 랑 완전히 동일하지만 BETWEEN 이 더 짧고 읽기 편해서 씀.

### IN — 다중 값 검색

여러 개의 값 중 하나라도 일치하면 반환. OR를 여러 번 쓰는 것과 동일하지만 더 간결함.

| 특징 | 설명 |
|------|------|
| 동일 표현 | `= 값1 OR = 값2 OR = 값3` 과 완전히 동일 |
| 순서 | 상관없음 |
| NULL | NULL 값은 매칭 안 됨 |

1) 기본 사용
```sql
SELECT * FROM [테이블명] WHERE profession IN ('Student', 'Baker', 'Programmer');

-- [결과] 셋 중 하나라도 일치하면 반환
-- ('alice', 'Student'), ('bob', 'Baker'), ('carol', 'Programmer')
-- ('dave', 'Chef') ❌ → 목록에 없음
```

2) OR 대신 IN — 가독성 비교
```sql
-- OR 사용 (길고 반복적)
SELECT * FROM [테이블명]
WHERE profession = 'Student' OR profession = 'Baker' OR profession = 'Programmer';

-- IN 사용 (간결)
SELECT * FROM [테이블명]
WHERE profession IN ('Student', 'Baker', 'Programmer');
```

> 값이 많아질수록 OR보다 IN이 훨씬 읽기 편함.

### 비교 연산자

| 연산자 | 의미 |
|--------|------|
| `=` | 같다 |
| `<>` | 같지 않다 (표준) |
| `!=` | 같지 않다 (비표준, 대부분 지원) |
| `>` / `>=` | 크다 / 크거나 같다 |
| `<` / `<=` | 작다 / 작거나 같다 |

```sql
-- 나이가 정확히 20살
SELECT * FROM users WHERE age = 20;

-- 탈퇴한 사용자 제외
SELECT * FROM users WHERE status <> 'inactive';

-- 점수가 80점 초과
SELECT * FROM exam WHERE score > 80;

-- 재고가 1개 이상인 상품
SELECT * FROM products WHERE stock >= 1;

-- 가격이 10000원 미만
SELECT * FROM products WHERE price < 10000;

-- 실무 조합: 활성 유저 중 성인만
SELECT * FROM users WHERE status = 'active' AND age >= 18;
```

## 3. 고급 SELECT

### ORDER BY — 정렬

조회 결과를 특정 컬럼 기준으로 정렬해서 반환.

| 특징 | 설명 |
|------|------|
| 기본값 | 오름차순 (`ASC`) — 생략 가능 |
| 내림차순 | `DESC` 명시 필요 |
| 여러 컬럼 | 왼쪽 컬럼 먼저 정렬, 같은 값이면 다음 컬럼으로 정렬 |
| NULL | 오름차순 시 NULL이 마지막에 위치 |

1) 오름차순 (기본)
```sql
SELECT * FROM [테이블명] ORDER BY age;

-- [결과] 작은 값부터
-- ('dave', 19), ('bob', 20), ('carol', 22), ('alice', 25)
```

2) 내림차순
```sql
SELECT * FROM [테이블명] ORDER BY age DESC;

-- [결과] 큰 값부터
-- ('alice', 25), ('carol', 22), ('bob', 20), ('dave', 19)
```

3) 여러 컬럼 정렬
```sql
SELECT * FROM [테이블명] ORDER BY age, name;

-- [결과] age 먼저 정렬, age 같으면 name으로 정렬
-- ('dave', 19), ('bob', 20), ('alice', 22), ('carol', 22)
-- age가 22로 같은 alice, carol은 name 알파벳 순으로 정렬됨
```

4) 혼합 정렬
```sql
SELECT * FROM [테이블명] ORDER BY age DESC, name ASC;

-- [결과] age 내림차순, age 같으면 name 오름차순
-- ('alice', 25), ('carol', 22), ('alice', 22), ('bob', 20)
```

### DISTINCT — 중복 제거

조회 결과에서 중복된 값을 제거하고 한 번만 출력.

```sql
SELECT DISTINCT profession FROM my_contacts;

-- [원본 데이터]
-- ('alice', 'Student'), ('bob', 'Baker'), ('carol', 'Student'),
-- ('dave', 'Programmer'), ('eve', 'Student'), ('frank', 'Baker')

-- [DISTINCT 적용 결과] 중복 제거 후 유니크한 값만
-- ('Student'), ('Baker'), ('Programmer')
-- Student 3명, Baker 2명이어도 각각 한 번만 출력됨
```

### 집계 함수

여러 행을 하나의 값으로 요약. NULL은 기본적으로 제외하고 계산.

| 함수 | 설명 |
|------|------|
| `MIN` | 최솟값 |
| `MAX` | 최댓값 |
| `AVG` | 평균 (NULL 제외) |
| `SUM` | 합계 (NULL 제외) |
| `COUNT(컬럼)` | NULL 제외한 행 수 |
| `COUNT(*)` | 전체 행 수 (NULL 포함) |

```sql
SELECT MIN(price) FROM store_sales;
SELECT MAX(price) FROM store_sales;
SELECT AVG(price) FROM store_sales;
SELECT SUM(price * quantity) FROM store_sales;  -- NULL이면 해당 행 제외

SELECT COUNT(price) FROM store_sales;   -- NULL 제외하고 카운트
SELECT COUNT(*) FROM store_sales;       -- 전체 행 카운트 (NULL 포함)

-- [결과 예시]
-- MIN → 500
-- MAX → 99000
-- AVG → 35000.00
-- COUNT(price) → 8  (NULL인 행 제외)
-- COUNT(*) → 10     (전체 행)
```

### COALESCE — NULL 대체

값이 NULL일 때 대체값으로 치환. NULL이 집계 결과를 왜곡할 때 자주 사용.

```sql
SELECT COALESCE(price, 0) FROM store_sales;
-- price가 NULL이면 0으로 대체

-- [결과]
-- 500, 0, 3000, 0, 99000
-- NULL이었던 값이 0으로 치환됨

-- NULL 제외 평균 vs 0으로 치환한 평균 차이
SELECT AVG(price) FROM store_sales;              -- NULL 제외하고 계산
SELECT AVG(COALESCE(price, 0)) FROM store_sales; -- NULL을 0으로 포함해서 계산
```

### NULLIF — 특정 값을 NULL로

두 값이 같으면 NULL 반환, 다르면 첫 번째 값 반환.

```sql
SELECT NULLIF(5, 5);  -- NULL 반환 (같음)
SELECT NULLIF(5, 3);  -- 5 반환 (다름)

-- 0으로 나누기 방지 (quantity가 0이면 NULL 반환 → 에러 방지)
SELECT price / NULLIF(quantity, 0) FROM store_sales;

-- COALESCE와 조합: price가 0이면 999로 처리
SELECT COALESCE(NULLIF(price, 0), 999) FROM store_sales;
-- price = 0  → NULLIF → NULL → COALESCE → 999
-- price = 500 → NULLIF → 500  → COALESCE → 500
```

### GROUP BY — 그룹 집계

공통된 값끼리 묶어서 집계. SUM, AVG, COUNT, MAX, MIN과 함께 사용.

```sql
SELECT status, COUNT(*) FROM orders GROUP BY status;

-- [결과] status별 주문 수
-- ('PAID', 5), ('PENDING', 3), ('CANCEL', 2)
```

```sql
SELECT category, COUNT(*), AVG(price) FROM store_sales GROUP BY category;

-- [결과] 카테고리별 상품 수와 평균 가격
-- ('food', 4, 3500.00), ('electronics', 6, 55000.00)
```

### HAVING — 그룹 필터링

GROUP BY 이후 그룹 조건 필터링. WHERE는 행 단위, HAVING은 그룹 단위.

| 구분 | 적용 시점 | 대상 |
|------|----------|------|
| `WHERE` | GROUP BY 이전 | 개별 행 |
| `HAVING` | GROUP BY 이후 | 그룹 |

```sql
SELECT main, SUM(amount) FROM drinks
GROUP BY main
HAVING SUM(amount) >= 3.0;

-- 1. main 컬럼으로 그룹화
-- 2. 그룹별 합계 계산
-- 3. 합계가 3.0 이상인 그룹만 반환

-- [결과]
-- ('juice', 4.5), ('soda', 3.0)
-- ('water', 1.5) ❌ → 합계가 3.0 미만
```

## 4. 데이터 수정 / 삭제

### DELETE

조건에 맞는 행을 삭제. 조건 없으면 전체 삭제되니 주의.

```sql
DELETE FROM [테이블명] WHERE [조건];

-- [예시] 연체일이 0 이하인 기록 삭제
DELETE FROM overdue_records WHERE overdue_days <= 0;

-- [주의] 조건 없으면 전체 삭제
DELETE FROM overdue_records;
```

### UPDATE

조건에 맞는 행의 값을 수정. 조건 없으면 전체 변경되니 주의.

```sql
UPDATE [테이블명] SET [컬럼명] = [값] WHERE [조건];

-- [예시] 특정 주문 상태 변경
UPDATE orders SET status = 'PAID' WHERE order_id IN (101, 103);

-- [주의] 조건 없으면 모든 행이 변경됨
UPDATE orders SET status = 'PAID';
```

## 5. ON CONFLICT (충돌 처리)

### DO NOTHING — 중복 무시

```sql
INSERT INTO newsletter_users(email)
VALUES ('alice@example.com')
ON CONFLICT (email) DO NOTHING;
-- 이미 존재하면 에러 없이 무시
```

대량 삽입 시:
```sql
INSERT INTO newsletter_users(email)
SELECT email FROM staging_list
ON CONFLICT (email) DO NOTHING;
```

성공한 행만 반환:
```sql
INSERT INTO newsletter_users(email)
SELECT email FROM staging_list
ON CONFLICT (email) DO NOTHING
RETURNING email;
```

### DO UPDATE — UPSERT (없으면 INSERT, 있으면 UPDATE)

```sql
INSERT INTO products (product_id, product_name, stock)
VALUES (1, 'macbook', 5)
ON CONFLICT (product_id) DO UPDATE
SET stock = products.stock + EXCLUDED.stock;
-- product_id=1이 없으면 새로 INSERT
-- 있으면 기존 재고 + 새 입고량으로 UPDATE
-- EXCLUDED : INSERT하려고 했던 값
```

| 상황 | 문법 |
|------|------|
| 이미 있으면 무시 | `ON CONFLICT DO NOTHING` |
| 이미 있으면 갱신 | `ON CONFLICT DO UPDATE` |

### WHERE — 조건 필터링

조회할 데이터의 조건을 지정. 조건에 맞는 행만 반환.

1) 단순 조건
```sql
SELECT * FROM users WHERE status = 'active';

-- [결과]
-- ('alice', 'active'), ('bob', 'active')
```

2) OR — 조건 중 하나라도 맞으면
```sql
SELECT * FROM users WHERE status = 'active' OR status = 'pending';

-- [결과]
-- ('alice', 'active'), ('carol', 'pending')
```

3) AND — 조건 둘 다 맞아야
```sql
SELECT * FROM users WHERE status = 'active' AND age >= 20;

-- [결과]
-- ('alice', 'active', 25)
```

4) 괄호로 우선순위 묶기
```sql
SELECT * FROM users WHERE (status = 'active' OR status = 'pending') AND age >= 20;

-- [결과]
-- ('alice', 'active', 25), ('carol', 'pending', 22)
```

### LIKE — 패턴 검색

문자열에서 특정 패턴을 찾는 검색.

| 와일드카드 | 의미 | 상세 |
|-----------|------|------|
| `%` | 0개 이상의 임의 문자 | **위치**만 맞으면 됨. 개수 자유 |
| `_` | 정확히 1개의 임의 문자 | **위치 + 개수** 둘 다 정확히 맞아야 함 |

1) `%` — **위치**가 핵심 (개수 상관없음)
```sql
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE 'a%';  -- a로 시작
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE '%a';  -- a로 끝남
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE '%a%'; -- a가 포함
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE '%CA'; -- CA로 끝나는 값

-- [결과] LIKE 'a%'
-- ('alice'), ('a'), ('abcdefg')
-- ('balice') ❌ → % 앞 a가 없음

-- [결과] LIKE '%CA' → % 위치가 앞이니까 앞에 뭐가 몇 개 오든 상관없음
-- ('San Francisco, CA'), ('Los Angeles, CA'), ('San Diego, CA')
```

2) `_` — **위치 + 개수** 둘 다 정확히 맞아야 함
```sql
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE 'a_';       -- a 뒤에 딱 1글자
SELECT * FROM [테이블명] WHERE [컬럼명] LIKE '_____@%';  -- @ 앞에 정확히 5글자

-- [결과] LIKE 'a_'
-- ('ab'), ('a1')
-- ('alice') ❌ → _ 1개인데 뒤에 4글자 / ('a') ❌ → _ 1개인데 뒤에 0글자

-- [결과] LIKE '_____@%' → _ 5개니까 @ 앞 글자 수가 정확히 5개여야 함, @ 뒤는 % 라서 상관없음
-- ('alice@gmail.com'), ('boost@naver.com'), ('hello@kakao.com')
-- ('al@gmail.com') ❌ → @ 앞이 2글자 / ('123456@gmail.com') ❌ → @ 앞이 6글자
```

### BETWEEN — 범위 검색

특정 범위 안에 있는 데이터를 찾는 검색.

| 특징 | 설명 |
|------|------|
| 끝값 | A, B 자기 자신 포함 (`>=`, `<=`) |
| 타입 | 숫자, 날짜, 문자열 |
| NULL | NULL 값은 범위에서 제외됨 |

1) 숫자 범위
```sql
SELECT * FROM [테이블명] WHERE age BETWEEN 20 AND 30;

-- [결과] 20 이상 30 이하 (양 끝값 포함)
-- ('alice', 25), ('bob', 20), ('carol', 30)
-- ('dave', 19) ❌ → 20 미만 / ('eve', 31) ❌ → 30 초과
```

2) 날짜 범위
```sql
SELECT * FROM [테이블명] WHERE birthday BETWEEN '1975-01-01' AND '1980-12-31';

-- [결과] 1975년 1월 1일 ~ 1980년 12월 31일 사이
-- ('1975-01-01'), ('1978-06-15'), ('1980-12-31')
-- ('1974-12-31') ❌ → 범위 이전 / ('1981-01-01') ❌ → 범위 이후
```

3) 문자열 범위
```sql
SELECT * FROM [테이블명] WHERE name BETWEEN 'A' AND 'M';

-- [결과] 알파벳 순서로 A ~ M 사이
-- ('Alice'), ('Bob'), ('Dave'), ('Mike')
-- ('Nathan') ❌ → N은 M 이후 / ('Zoe') ❌ → Z는 M 이후
```

> `>= A AND <= B` 랑 완전히 동일하지만 BETWEEN 이 더 짧고 읽기 편해서 씀.

### IN — 다중 값 검색

여러 개의 값 중 하나라도 일치하면 반환. OR를 여러 번 쓰는 것과 동일하지만 더 간결함.

| 특징 | 설명 |
|------|------|
| 동일 표현 | `= 값1 OR = 값2 OR = 값3` 과 완전히 동일 |
| 순서 | 상관없음 |
| NULL | NULL 값은 매칭 안 됨 |

1) 기본 사용
```sql
SELECT * FROM [테이블명] WHERE profession IN ('Student', 'Baker', 'Programmer');

-- [결과] 셋 중 하나라도 일치하면 반환
-- ('alice', 'Student'), ('bob', 'Baker'), ('carol', 'Programmer')
-- ('dave', 'Chef') ❌ → 목록에 없음
```

2) OR 대신 IN — 가독성 비교
```sql
-- OR 사용 (길고 반복적)
SELECT * FROM [테이블명]
WHERE profession = 'Student' OR profession = 'Baker' OR profession = 'Programmer';

-- IN 사용 (간결)
SELECT * FROM [테이블명]
WHERE profession IN ('Student', 'Baker', 'Programmer');
```

> 값이 많아질수록 OR보다 IN이 훨씬 읽기 편함.

### 비교 연산자

| 연산자 | 의미 |
|--------|------|
| `=` | 같다 |
| `<>` | 같지 않다 (표준) |
| `!=` | 같지 않다 (비표준, 대부분 지원) |
| `>` / `>=` | 크다 / 크거나 같다 |
| `<` / `<=` | 작다 / 작거나 같다 |

## 3. 고급 SELECT

### ORDER BY — 정렬

조회 결과를 특정 컬럼 기준으로 정렬해서 반환.

| 특징 | 설명 |
|------|------|
| 기본값 | 오름차순 (`ASC`) — 생략 가능 |
| 내림차순 | `DESC` 명시 필요 |
| 여러 컬럼 | 왼쪽 컬럼 먼저 정렬, 같은 값이면 다음 컬럼으로 정렬 |
| NULL | 오름차순 시 NULL이 마지막에 위치 |

1) 오름차순 (기본)
```sql
SELECT * FROM [테이블명] ORDER BY age;

-- [결과] 작은 값부터
-- ('dave', 19), ('bob', 20), ('carol', 22), ('alice', 25)
```

2) 내림차순
```sql
SELECT * FROM [테이블명] ORDER BY age DESC;

-- [결과] 큰 값부터
-- ('alice', 25), ('carol', 22), ('bob', 20), ('dave', 19)
```

3) 여러 컬럼 정렬
```sql
SELECT * FROM [테이블명] ORDER BY age, name;

-- [결과] age 먼저 정렬, age 같으면 name으로 정렬
-- ('dave', 19), ('bob', 20), ('alice', 22), ('carol', 22)
-- age가 22로 같은 alice, carol은 name 알파벳 순으로 정렬됨
```

4) 혼합 정렬
```sql
SELECT * FROM [테이블명] ORDER BY age DESC, name ASC;

-- [결과] age 내림차순, age 같으면 name 오름차순
-- ('alice', 25), ('carol', 22), ('alice', 22), ('bob', 20)
```

### DISTINCT — 중복 제거

조회 결과에서 중복된 값을 제거하고 한 번만 출력.

```sql
SELECT DISTINCT profession FROM my_contacts;

-- [결과] 중복 제거 후 유니크한 값만
-- ('Student'), ('Baker'), ('Programmer')
-- 원본에 Student가 5명이어도 한 번만 출력됨
```

### 집계 함수

여러 행을 하나의 값으로 요약. NULL은 기본적으로 제외하고 계산.

| 함수 | 설명 |
|------|------|
| `MIN` | 최솟값 |
| `MAX` | 최댓값 |
| `AVG` | 평균 (NULL 제외) |
| `SUM` | 합계 (NULL 제외) |
| `COUNT(컬럼)` | NULL 제외한 행 수 |
| `COUNT(*)` | 전체 행 수 (NULL 포함) |

```sql
SELECT MIN(price) FROM store_sales;
SELECT MAX(price) FROM store_sales;
SELECT AVG(price) FROM store_sales;
SELECT SUM(price * quantity) FROM store_sales;  -- NULL이면 해당 행 제외

SELECT COUNT(price) FROM store_sales;   -- NULL 제외하고 카운트
SELECT COUNT(*) FROM store_sales;       -- 전체 행 카운트 (NULL 포함)

-- [결과 예시]
-- MIN → 500
-- MAX → 99000
-- AVG → 35000.00
-- COUNT(price) → 8  (NULL인 행 제외)
-- COUNT(*) → 10     (전체 행)
```

### COALESCE — NULL 대체

값이 NULL일 때 대체값으로 치환. NULL이 집계 결과를 왜곡할 때 자주 사용.

```sql
SELECT COALESCE(price, 0) FROM store_sales;
-- price가 NULL이면 0으로 대체

-- [결과]
-- 500, 0, 3000, 0, 99000
-- NULL이었던 값이 0으로 치환됨

-- NULL 제외 평균 vs 0으로 치환한 평균 차이
SELECT AVG(price) FROM store_sales;              -- NULL 제외하고 계산
SELECT AVG(COALESCE(price, 0)) FROM store_sales; -- NULL을 0으로 포함해서 계산
```

### NULLIF — 특정 값을 NULL로

두 값이 같으면 NULL 반환, 다르면 첫 번째 값 반환.

```sql
SELECT NULLIF(5, 5);  -- NULL 반환 (같음)
SELECT NULLIF(5, 3);  -- 5 반환 (다름)

-- 0으로 나누기 방지 (quantity가 0이면 NULL 반환 → 에러 방지)
SELECT price / NULLIF(quantity, 0) FROM store_sales;

-- COALESCE와 조합: price가 0이면 999로 처리
SELECT COALESCE(NULLIF(price, 0), 999) FROM store_sales;
-- price = 0  → NULLIF → NULL → COALESCE → 999
-- price = 500 → NULLIF → 500  → COALESCE → 500
```

### GROUP BY — 그룹 집계

공통된 값끼리 묶어서 집계. SUM, AVG, COUNT, MAX, MIN과 함께 사용.

```sql
SELECT status, COUNT(*) FROM orders GROUP BY status;

-- [결과] status별 주문 수
-- ('PAID', 5), ('PENDING', 3), ('CANCEL', 2)
```

```sql
SELECT category, COUNT(*), AVG(price) FROM store_sales GROUP BY category;

-- [결과] 카테고리별 상품 수와 평균 가격
-- ('food', 4, 3500.00), ('electronics', 6, 55000.00)
```

### HAVING — 그룹 필터링

GROUP BY 이후 그룹 조건 필터링. WHERE는 행 단위, HAVING은 그룹 단위.

| 구분 | 적용 시점 | 대상 |
|------|----------|------|
| `WHERE` | GROUP BY 이전 | 개별 행 |
| `HAVING` | GROUP BY 이후 | 그룹 |

```sql
SELECT main, SUM(amount) FROM drinks
GROUP BY main
HAVING SUM(amount) >= 3.0;

-- 1. main 컬럼으로 그룹화
-- 2. 그룹별 합계 계산
-- 3. 합계가 3.0 이상인 그룹만 반환

-- [결과]
-- ('juice', 4.5), ('soda', 3.0)
-- ('water', 1.5) ❌ → 합계가 3.0 미만
```

## 4. 데이터 수정 / 삭제

### DELETE

조건에 맞는 행을 삭제. 조건 없으면 전체 삭제되니 주의.

```sql
DELETE FROM [테이블명] WHERE [조건];

-- [예시] 연체일이 0 이하인 기록 삭제
DELETE FROM overdue_records WHERE overdue_days <= 0;

-- [주의] 조건 없으면 전체 삭제
DELETE FROM overdue_records;
```

### UPDATE

조건에 맞는 행의 값을 수정. 조건 없으면 전체 변경되니 주의.

```sql
UPDATE [테이블명] SET [컬럼명] = [값] WHERE [조건];

-- [예시] 특정 주문 상태 변경
UPDATE orders SET status = 'PAID' WHERE order_id IN (101, 103);

-- [주의] 조건 없으면 모든 행이 변경됨
UPDATE orders SET status = 'PAID';
```

## 5. ON CONFLICT (충돌 처리)

### DO NOTHING — 중복 무시

```sql
INSERT INTO newsletter_users(email)
VALUES ('alice@example.com')
ON CONFLICT (email) DO NOTHING;
-- 이미 존재하면 에러 없이 무시
```

대량 삽입 시:
```sql
INSERT INTO newsletter_users(email)
SELECT email FROM staging_list
ON CONFLICT (email) DO NOTHING;
```

성공한 행만 반환:
```sql
INSERT INTO newsletter_users(email)
SELECT email FROM staging_list
ON CONFLICT (email) DO NOTHING
RETURNING email;
```

### DO UPDATE — UPSERT (없으면 INSERT, 있으면 UPDATE)

```sql
INSERT INTO products (product_id, product_name, stock)
VALUES (1, 'macbook', 5)
ON CONFLICT (product_id) DO UPDATE
SET stock = products.stock + EXCLUDED.stock;
-- product_id=1이 없으면 새로 INSERT
-- 있으면 기존 재고 + 새 입고량으로 UPDATE
-- EXCLUDED : INSERT하려고 했던 값
```

| 상황 | 문법 |
|------|------|
| 이미 있으면 무시 | `ON CONFLICT DO NOTHING` |
| 이미 있으면 갱신 | `ON CONFLICT DO UPDATE` |

## 핵심 요약

```
DDL  : CREATE, DROP (구조)
DML  : INSERT, SELECT, UPDATE, DELETE (데이터)

SELECT 절 순서
  SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY

WHERE 조건
  LIKE    : 패턴 검색 (% 위치, _ 위치+개수)
  BETWEEN : 범위 검색, 양 끝값 포함
  IN      : 다중 값 검색, OR 대신 사용

ON CONFLICT
  DO NOTHING : 중복 무시
  DO UPDATE  : UPSERT (없으면 INSERT, 있으면 UPDATE)
```
