### 3) E-R 다이어그램으로 표현

> STEP 1, 2의 결과물(개체 + 관계)을 시각화한 최종 산출물
>
> DBMS 종류에 관계없이 동일한 표기법 사용

**표기법**

| 기호 | 의미 |
|---|---|
| `[ ]` 사각형 | 개체 (Entity) |
| `< >` 마름모 | 관계 (Relationship) |
| 타원 | 속성 (Attribute) |
| 실선 | 필수적 참여 |
| 점선 | 선택적 참여 |

**예시**

```
[환자]──────<진료>──────[의사]
  |          N:M           |
환자번호               면허번호
이름                   이름
생년월일               전공
전화번호
  |
  | 1:N
  |
[처방전]
처방전번호
발급일
```

> - `환자 - 의사`: N:M 관계 → 나중에 릴레이션 변환 시 중간 테이블 생성
> - `의사 - 처방전`: 1:N 관계 → 처방전 테이블에 FK 추가
> - `환자 - 처방전`: 1:N 관계 → 처방전 테이블에 FK 추가

**결과물:** 개념적 스키마 (E-R 다이어그램)
