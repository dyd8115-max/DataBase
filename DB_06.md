# DB | Relational Algebra (관계 대수)


---

## 1. 관계 대수란?

**관계 대수(Relational Algebra)**란 주어진 릴레이션(테이블)으로부터 원하는 결과를 얻기 위해 **연산 순서를 명시하는 절차적 형식 언어**다.

| 구분 | 설명 |
|------|------|
| **형식 언어 (Formal Language)** | 문법(syntax)과 의미(semantics)가 수학적으로 엄밀하게 정의된 언어 |
| **절차적 언어 (Procedural Language)** | 원하는 결과를 얻기 위한 **연산 절차(순서)**를 명시 |
| **폐쇄 특성 (Closure Property)** | 연산의 입력도 릴레이션, 출력도 릴레이션 → 연산 결과를 다시 연산에 활용 가능 |

>  관계 대수는 SQL의 이론적 기반이다. DBMS는 SQL을 파싱해 내부적으로 관계 대수식으로 변환한 뒤 최적화하여 실행한다.

**관계 대수 연산의 3가지 핵심 기능**

| 기능 | 설명 |
|------|------|
| **필터링 (Filter)** | 조건을 만족하는 행(튜플)만 추출 |
| **슬라이싱 (Slice)** | 원하는 열(속성)만 추출 |
| **결합 (Combine)** | 여러 릴레이션을 하나로 합침 |

---

## 2. 쿼리 처리 과정

```
SQL 쿼리 입력
     ↓
1. 파싱 & 변환 (Parsing & Translation)
   - 문법 검사, 릴레이션 존재 여부 확인
   - SQL → 관계 대수식으로 변환
     ↓
2. 최적화 (Optimization)
   - 동일한 결과를 내는 다양한 관계 대수식 중
     가장 효율적인 실행 계획 선택
     ↓
3. 실행 (Evaluation)
   - 선택된 실행 계획으로 실제 연산 수행
```

>  **왜 최적화가 필요한가?**  
> 하나의 SQL은 동일한 결과를 내는 **여러 관계 대수식**으로 표현 가능하다.  
> 예: `σ조건(R ⋈ S)` 와 `σ조건(R) ⋈ S` 는 결과가 같지만 성능이 다르다.  
> → 조건을 먼저 적용해 데이터 크기를 줄이는 것이 일반적으로 더 빠르다.

---

## 3. 연산자 분류

관계 대수의 연산자는 **기본 연산자**와 **유도 연산자**로 나뉜다.

| 구분 | 연산자 | 기호 | 설명 |
|------|--------|------|------|
| **기본 연산자** (Primitive) | 셀렉트 | σ | 조건을 만족하는 행 추출 |
| | 프로젝트 | π | 지정한 열만 추출 |
| | 합집합 | ∪ | 두 릴레이션의 합 |
| | 차집합 | − | 두 릴레이션의 차 |
| | 카티션 곱 | × | 두 릴레이션의 모든 조합 |
| **유도 연산자** (Derived) | 조인 | ⋈ | 기본 연산자의 조합 |
| | 교집합 | ∩ | 기본 연산자의 조합 |
| | 나눗셈 | ÷ | 기본 연산자의 조합 |
| | 이름 변경 | ρ | 릴레이션/속성 이름 변경 |

>  유도 연산자는 기본 연산자만으로 표현 가능하지만, **자주 사용되는 패턴을 편의상 별도 연산자로 정의**한 것이다.

## 4. 집합 연산 (Set Operations)

집합 연산은 수학의 집합 이론을 릴레이션에 적용한 연산이다.

### 합집합 호환 조건 (Union-Compatible)

합집합(`∪`), 교집합(`∩`), 차집합(`−`) 연산을 수행하려면 두 릴레이션이 반드시 **합집합 호환** 조건을 만족해야 한다.

> **합집합 호환 조건**
> 1. 두 릴레이션의 **차수(속성 개수)가 동일**해야 한다.
> 2. 대응하는 속성끼리 **도메인(값의 종류)이 동일**해야 한다.

---

### 4.1 합집합 (Union)

**기호**: `R ∪ S`

> R에 있거나 S에 있는 **모든 튜플**을 반환한다. 중복 튜플은 제거된다.

```
R ∪ S = S ∪ R              (교환 법칙)
(R ∪ S) ∪ T = R ∪ (S ∪ T) (결합 법칙)
```

**결과 릴레이션의 특성**
- 차수: R과 S의 차수와 동일
- 카디널리티: R과 S의 카디널리티의 합 이하 (중복 제거)

```
R:          S:          R ∪ S:
A  | B      A  | B      A  | B
---|---      ---|---      ---|---
1  | a      1  | a      1  | a
2  | b      3  | c      2  | b
                        3  | c
```

---

### 4.2 교집합 (Intersection)

**기호**: `R ∩ S`

> R과 S **모두에 있는 튜플**만 반환한다.

```
R ∩ S = S ∩ R              (교환 법칙)
(R ∩ S) ∩ T = R ∩ (S ∩ T) (결합 법칙)
```

**결과 릴레이션의 특성**
- 차수: R과 S의 차수와 동일
- 카디널리티: R과 S 중 작은 카디널리티 이하

```
R:          S:          R ∩ S:
A  | B      A  | B      A  | B
---|---      ---|---      ---|---
1  | a      1  | a      1  | a
2  | b      3  | c
```

---

### 4.3 차집합 (Difference)

**기호**: `R − S`

> R에는 있지만 **S에는 없는 튜플**만 반환한다.

> ⚠️ 차집합은 **교환·결합 법칙이 성립하지 않는다.**  
> `R − S ≠ S − R`

**결과 릴레이션의 특성**
- 차수: R과 S의 차수와 동일
- `R − S`의 카디널리티: R의 카디널리티 이하
- `S − R`의 카디널리티: S의 카디널리티 이하

```
R:          S:          R − S:      S − R:
A  | B      A  | B      A  | B      A  | B
---|---      ---|---      ---|---      ---|---
1  | a      1  | a      2  | b      3  | c
2  | b      3  | c
```

---

### 4.4 카티션 곱 (Cartesian Product)

**기호**: `R × S`

> R의 모든 튜플과 S의 모든 튜플을 **가능한 모든 조합**으로 이어 붙인 릴레이션을 반환한다.

```
R × S = S × R              (교환 법칙)
(R × S) × T = R × (S × T) (결합 법칙)
```

**결과 릴레이션의 특성**
- 차수: R의 차수 + S의 차수
- 카디널리티: R의 카디널리티 × S의 카디널리티

```
R:        S:        R × S:
A  | B    C  | D    A  | B  | C  | D
---|---    ---|---    ---|---|---|---
1  | a    x  | 10   1  | a | x | 10
2  | b    y  | 20   1  | a | y | 20
                    2  | b | x | 10
                    2  | b | y | 20
```

>  카티션 곱만으로는 의미 있는 결과를 얻기 어렵다. 보통 **셀렉트(σ)와 함께 사용**하며, 이것이 조인의 기반이 된다.

## 5. 관계 연산 (Relational Operations)

릴레이션의 구조적 특성을 활용하는 연산들이다.

---

### 5.1 셀렉트 (Select, σ)

**기호**: `σ조건(릴레이션)`  
**SQL 대응**: `WHERE 절`

> 릴레이션에서 **주어진 조건을 만족하는 튜플(행)**만 추출한다.  
> → 결과 릴레이션의 **차수는 동일**, **카디널리티는 감소**한다.

**조건(선택 조건/술어)**
- 비교 연산자: `>`, `≥`, `<`, `≤`, `=`, `≠`
- 논리 연산자: `∧` (AND), `∨` (OR), `¬` (NOT)

```
표기법: σ조건(릴레이션)
SQL:    SELECT * FROM 릴레이션 WHERE 조건;
```

**예시**: 학생 릴레이션에서 등급이 'gold'이고 잔고가 2000 초과인 학생 조회

```
σ등급='gold' ∧ 잔고>2000(학생)

SQL: SELECT * FROM 학생
     WHERE 등급 = 'gold' AND 잔고 > 2000;
```

**셀렉트의 특성**
```
교환 법칙: σ조건1(σ조건2(R)) = σ조건2(σ조건1(R))
결합 법칙: σ조건1 ∧ 조건2(R) = σ조건1(σ조건2(R))
```

---

### 5.2 프로젝트 (Project, π)

**기호**: `π속성목록(릴레이션)`  
**SQL 대응**: `SELECT 열목록`

> 릴레이션에서 **지정한 속성(열)만** 추출한다.  
> → 결과 릴레이션의 **카디널리티는 동일**, **차수는 감소**.  
> → 중복 튜플은 **자동 제거**된다.

```
표기법: π속성목록(릴레이션)
SQL:    SELECT 속성1, 속성2 FROM 릴레이션;
```

**예시**: 학생 릴레이션에서 이름과 학과만 조회

```
π이름, 학과(학생)

SQL: SELECT 이름, 학과 FROM 학생;
```

> ⚠️ 프로젝트 결과에서 **중복 튜플이 제거**됨에 주의.  
> 예: 학생 테이블에서 `π학과`를 수행하면 같은 학과 학생이 여럿이어도 각 학과는 한 번만 등장한다.

---

### 5.3 자연 조인 (Natural Join, ⋈)

**기호**: `R ⋈ S` 또는 `R ⋈N S`

> 두 릴레이션에서 **공통 속성(이름과 도메인이 같은 속성)** 의 값이 일치하는 튜플을 결합한다.  
> → 공통 속성은 결과에서 **한 번만** 등장한다.

**자연 조인 = Cartesian Product + Selection + Projection**
```
R ⋈ S = π중복제거(σR.공통속성=S.공통속성(R × S))
```

```
예시: R(A, B, C, D), S(B, D, E)
     공통 속성: B, D
     결과: R ⋈ S (A, B, C, D, E) — B와 D가 일치하는 행만, B/D 중복 없음
```

**예시**: 학생 ⋈ 수강 (공통 속성: 학번)

```
학생(학번, 이름, 학과) ⋈ 수강(학번, 과목코드, 성적)
→ 결과: (학번, 이름, 학과, 과목코드, 성적)
   — 두 테이블에서 학번이 일치하는 행만 포함
```

> ⚠️ 공통 속성의 **이름이 다를 경우** 자연 조인이 적용되지 않는다.  
> 예: 학생 테이블의 `학번`과 수강 테이블의 `student_id`는 이름이 달라 자연 조인 불가 → 이 경우 **동등 조인** 사용

---

### 5.4 동등 조인 (Equi-Join)

**기호**: `R ⋈R.속성=S.속성 S`

> 두 릴레이션에서 **지정한 속성의 값이 같은(=) 튜플**을 결합한다.  
> 자연 조인과 달리 **공통 속성이 결과에 두 번** 등장한다.

```
Equi-Join = Cartesian Product + Selection (= 조건)
자연 조인  = Equi-Join + Projection (중복 속성 제거)
```

```
학생 ⋈학생.학번=수강.학번 수강
→ 결과: (학생.학번, 이름, 학과, 수강.학번, 과목코드, 성적)
         ↑ 학번이 두 번 나타남
```

| 비교 | 자연 조인 | 동등 조인 |
|------|----------|---------|
| 조인 조건 | 공통 속성 이름 자동 매칭 | 속성 명시 (`R.A = S.B`) |
| 공통 속성 | 결과에 1번 | 결과에 2번 |
| 속성 이름 | 동일해야 함 | 달라도 가능 |

---

### 5.5 세타 조인 (Theta-Join)

**기호**: `R ⋈A θ B S`  
(θ는 비교 연산자: `>`, `≥`, `<`, `≤`, `=`, `≠`)

> 동등 조인을 일반화한 것으로, **등호(=) 외의 비교 연산자**도 사용 가능한 조인이다.  
> 두 릴레이션에서 지정한 조건을 만족하는 모든 튜플 조합을 반환한다.

```
결과 릴레이션의 차수 = R의 차수 + S의 차수
```

>  동등 조인 = θ 연산자가 `=`인 세타 조인

**예시**: 직원(Name, Salary)와 등급표(MinSalary, Grade)에서 직원의 급여가 최소급여 이상인 등급 조회

```
Employee ⋈Employee.Salary≥Grade.MinSalary Grade

Name  | Salary | MinSalary | Grade
------|--------|-----------|------
Kim   | 3000   | 2000      | C
Lee   | 5000   | 2000      | C
Lee   | 5000   | 4000      | B
```

#### 조인 종류 비교

```
세타 조인 (Theta-Join): R ⋈A θ B S  (θ = 임의 비교 연산자)
    ↑ θ가 '='일 때
동등 조인 (Equi-Join):  R ⋈A=B S     (공통 속성이 2번 등장)
    ↑ 중복 속성 제거
자연 조인 (Natural Join): R ⋈ S      (공통 속성이 1번 등장)
```

---

### 5.6 나눗셈 (Division, ÷)

**기호**: `R ÷ S`

> S의 **모든 튜플과 연관된** R의 튜플만 반환한다.  
> "S의 모든 값을 포함하는 R의 그룹"을 찾는 연산이다.

> ⚠️ 연산 적용 조건: R의 속성이 S의 속성을 **포함(superset)**해야 한다.

**예시**: "모든 과목을 수강한 학생"을 찾는 경우

```
수강(학번, 과목코드) ÷ 과목(과목코드)
→ 과목 테이블의 모든 과목코드를 수강한 학번만 결과에 포함
```

```
수강 R:              과목 S:     R ÷ S:
학번  | 과목코드       과목코드    학번
------|--------       --------   ----
1001  | CS101         CS101      1001  (CS101, CS102 모두 수강)
1001  | CS102         CS102
1002  | CS101
```

>  나눗셈은 SQL에서 직접 지원되지 않으므로, **NOT EXISTS** 등의 서브쿼리로 구현한다.

```sql
-- "모든 과목을 수강한 학생"을 SQL로 표현
SELECT DISTINCT 학번
FROM 수강 S1
WHERE NOT EXISTS (
    SELECT 과목코드 FROM 과목
    EXCEPT
    SELECT 과목코드 FROM 수강 S2
    WHERE S2.학번 = S1.학번
);
```

## 6. 확장 연산 (Extended Operations)

기본 연산자와 유도 연산자로 파생된 특수 목적 연산들이다.

---

### 6.1 세미 조인 (Semi-Join, ⋉)

**기호**: `R ⋉ S`

> S와 조인 가능한 튜플 중 **R의 속성만** 결과에 포함한다.  
> = 자연 조인 후 R의 속성만 프로젝트한 결과

```
R ⋉ S = πR의 속성(R ⋈ S)
```

**특성**
- 비교환적: `R ⋉ S ≠ S ⋉ R`
- S의 속성을 네트워크로 전송하지 않고 조인 가능 → **분산 데이터베이스 최적화**에 활용

**예시**: 학생 ⋉ 수강  
→ 수강 내역이 있는 학생 정보만 반환 (수강 테이블의 속성은 포함하지 않음)

```
학생(학번, 이름, 학과) ⋉ 수강(학번, 과목코드, 성적)
→ 결과: (학번, 이름, 학과) — 수강 레코드가 존재하는 학생만
```

---

### 6.2 외부 조인 (Outer-Join)

**문제점**: 자연 조인/내부 조인은 **매칭되지 않는 튜플을 결과에서 제외**한다.  
**해결**: 외부 조인은 매칭되지 않는 튜플도 결과에 포함하고, **없는 속성 값은 NULL**로 채운다.

| 종류 | 기호 | 설명 |
|------|------|------|
| **왼쪽 외부 조인** (Left Outer Join) | `R ⟕ S` | R의 모든 튜플 보존 |
| **오른쪽 외부 조인** (Right Outer Join) | `R ⟖ S` | S의 모든 튜플 보존 |
| **완전 외부 조인** (Full Outer Join) | `R ⟗ S` | R과 S의 모든 튜플 보존 |

#### 왼쪽 외부 조인 (Left Outer Join)

> R의 **모든 튜플**을 결과에 포함.  
> S에서 매칭되는 튜플이 없으면 S의 속성을 **NULL**로 채움.

```
학생(학번, 이름) ⟕ 수강(학번, 과목코드)

학번  | 이름  | 과목코드
------|-------|--------
1001  | Kim   | CS101
1001  | Kim   | CS102
1002  | Lee   | NULL   ← 수강 내역 없는 학생도 포함
```

#### 오른쪽 외부 조인 (Right Outer Join)

> S의 **모든 튜플**을 결과에 포함.  
> R에서 매칭되는 튜플이 없으면 R의 속성을 **NULL**로 채움.

```
학생(학번, 이름) ⟖ 수강(학번, 과목코드)

학번  | 이름  | 과목코드
------|-------|--------
1001  | Kim   | CS101
1001  | Kim   | CS102
NULL  | NULL  | CS103  ← 학생 정보 없는 수강 레코드도 포함
```

#### 완전 외부 조인 (Full Outer Join)

> R과 S의 **모든 튜플**을 결과에 포함.  
> 매칭되지 않는 쪽의 속성은 **NULL**로 채움.  
> = 왼쪽 외부 조인 ∪ 오른쪽 외부 조인

```
학생(학번, 이름) ⟗ 수강(학번, 과목코드)

학번  | 이름  | 과목코드
------|-------|--------
1001  | Kim   | CS101
1001  | Kim   | CS102
1002  | Lee   | NULL   ← 수강 없는 학생
NULL  | NULL  | CS103  ← 학생 없는 수강
```

#### 내부 조인 vs 외부 조인 비교

```
[학생]            [수강]
학번  이름         학번  과목코드
1001  Kim          1001  CS101
1002  Lee          1003  CS102   ← 학생 테이블에 없는 학번
1003  Park

INNER JOIN (자연 조인):
학번  이름   과목코드
1001  Kim    CS101
1003  Park   CS102   ← 1002 Lee는 수강 없어서 제외

LEFT OUTER JOIN:
학번  이름   과목코드
1001  Kim    CS101
1002  Lee    NULL    ← Lee 포함 (수강 없음)
1003  Park   CS102

RIGHT OUTER JOIN:
학번  이름   과목코드
1001  Kim    CS101
1003  Park   CS102
NULL  NULL   CS102   ← (1003 Park의 중복이 아닌, 매핑안된 케이스)

FULL OUTER JOIN:
학번  이름   과목코드
1001  Kim    CS101
1002  Lee    NULL
1003  Park   CS102
```

## 📝 핵심 요약

### 연산자 한눈에 보기

| 연산자 | 기호 | 입력 | 출력 특성 | 주요 용도 |
|--------|------|------|----------|----------|
| 셀렉트 | σ | 1개 릴레이션 | 행↓ 열= | 조건 필터링 |
| 프로젝트 | π | 1개 릴레이션 | 행= 열↓ (중복제거) | 열 선택 |
| 합집합 | ∪ | 2개 (호환 조건) | 행↑ | 합치기 |
| 교집합 | ∩ | 2개 (호환 조건) | 행↓ | 공통 추출 |
| 차집합 | − | 2개 (호환 조건) | 행↓ | 제외 |
| 카티션 곱 | × | 2개 | 행↑↑ 열↑ | 조인 기반 |
| 자연 조인 | ⋈ | 2개 | 공통속성 1번 | 관계 결합 |
| 동등 조인 | ⋈= | 2개 | 공통속성 2번 | 명시적 조인 |
| 세타 조인 | ⋈θ | 2개 | θ 조건 충족 | 범위 조인 |
| 나눗셈 | ÷ | 2개 | "모두 포함" | 전부 조건 |
| 세미 조인 | ⋉ | 2개 | R 속성만 | 분산 최적화 |
| 외부 조인 | ⟕⟖⟗ | 2개 | NULL 포함 | 누락 없는 결합 |

### 조인 계층 구조

```
세타 조인 (θ 임의)
   └─ 동등 조인 (θ = '=')
         └─ 자연 조인 (동등 조인 + 중복 속성 제거)

자연 조인 ⊂ 동등 조인 ⊂ 세타 조인
```

### 집합 연산 법칙 정리

| 연산 | 교환 법칙 | 결합 법칙 |
|------|---------|---------|
| 합집합 (∪) | ✅ | ✅ |
| 교집합 (∩) | ✅ | ✅ |
| 차집합 (−) | ❌ | ❌ |
| 카티션 곱 (×) | ✅ | ✅ |
| 세미 조인 (⋉) | ❌ | - |
