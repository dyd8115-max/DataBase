## 정규화의 개념과 이상현상


### 이상현상의 뜻과 종류

- 이상현상: 불필요한 데이터 중복으로 인해 릴레이션에 대한 데이터 삽입, 수정, 삭제 연산을 수행 할 떄 발생할 수 있는 부작용


1. 삽입현상:
   - 새 대이터를 삽입하기 위해 불필요한 데이터도 함께 삽입해야 하는 문제
   - 새 데이터를 삽입할 때 기존 속성 양식과 맞지 않으면 삽입 자체가 불가능 하여 임의의 값을 같이 넣음
   - ex) 신규 교수를 등록하기 위해 개설되지 않은 임의의 강의도 같이 넣음


2. 갱신이상:
   - 중복 투플 중 일부만 변경하여 데이터가 불일치하게 되는 모순의 문제
   - 여러 행에 데이터가 중복되어 입력되다 보니 미쳐 놓친 데이터 발생
   - ex) 강의를 드랍했으나 일부 페이지에서 드랍한 강의가 계속 보이는 현상

3. 삭제 이상:
   - 투플을 삭제하면 꼭 필요한 데이터까지 함께 삭제되는 데이터 손실의 문제
   - 하나의 테이블에 서로 다른 정보가 같이 저장되어 한 행을 지울 시 의도치 않은 정보 같이 삭제
   - ex) 고객의 주문 내역을 취소한 뒤 고객 id 자체가 사라짐

## 정규화 & 함수적 종속성


- 정규화:
  - 이상현상을 제거하면서 DB를 올바르게 설계해 나가는 과정
  - 릴레이션을 관련있는 속성들로만 구성하기 위해 릴레이션을 분해하는 과정
  - 함수적 종속성을 판단하여 정규화를 수행함

 
- 함수적 종속성(fd):
  - 두 개의 속성 집합들에 대한 제약 조건 (x->y) 
  - X 값이 같으면 Y 값도 반드시 같아야 함 (X가 Y를 유일하게 결정)
  - 릴레이션 내의 모든 투플에서 하나의 X값에 대한 Y값이 항상 하나임
  - X와 Y는 하나의 릴레이션을 구성하는 속성들의 부분집합
  - Y는 X에 함수적으로 종속됨
  - 이상현상 분석 도구
  - 단순 데이터만 보고 판단 x, 속성 자체의 특성과 의미를 기반으로 판단해야함
  - 일반적으로 기본키와 후보키는 릴레이션의 다른 모든 속성들을 함수적으로 결정함
  
  

### 함수적 종속성 예시

| 학번 | 이름 | 학과 |
|---|---|---|
| 20220001 | 김철수 | CS |
| 20220002 | 이영희 | EE |

```
학번 → 이름  ✅
학번 → 학과  ✅
이름 → 학번  ❌ (동명이인 가능)


학번 ->(이름, 학과)
```

 
### 함수적 종속성과 후보키와의 관계

- 후보키는 FD의 가장 강력한 형태
- 일반 FD는 일부 속성을 결정, 후보키는 릴레이션 전체를 결정

| | FD | 후보키 |
|---|---|---|
| 예시 | 학번 → 학과 | 학번 → R 전체 |
| 범위 | 일부 속성 결정 | 릴레이션 전체 결정 |
| 최소성 | 없어도 됨 | 반드시 만족 |

```
학번 → {이름, 학과, 나이, ...} = 학번 → R
∴ 학번은 후보키
```

> FD를 만족한다고 모두 후보키는 아님, 후보키는 전체 결정 + 최소성까지 만족해야 함


## 완전 함수적 종속 (Full Functional Dependency, FFD)

- X → Y 에서 X의 속성을 하나라도 제거하면 종속 관계가 깨지는 경우
- 즉, Y를 결정하기 위해 X 전체가 반드시 필요한 경우

### 예시

```
기본키: {학번, 이벤트번호}

{학번, 이벤트번호} → 당첨여부  ✅
- 학번만으로는 당첨여부 결정 불가
- 이벤트번호만으로는 당첨여부 결정 불가
- 둘 다 있어야 결정 가능 → 완전 함수적 종속
```

## 부분 함수적 종속 (Partial Functional Dependency, PFD)

- X → Y 에서 X의 일부만으로도 Y를 결정할 수 있는 경우
- 2NF 위반의 원인 → 2NF에서 제거 대상

### 예시

| 학번 | 이벤트번호 | 당첨여부 | 이름 |
|---|---|---|---|
| 20220001 | E1 | Y | 김철수 |
| 20220001 | E2 | N | 김철수 |

```
기본키: {학번, 이벤트번호}

{학번, 이벤트번호} → 당첨여부  ✅ 완전 함수적 종속
학번 → 이름                   ❌ 부분 함수적 종속
```

> 이름은 이벤트번호와 무관하게 학번만으로 결정됨
> → 키 전체가 아닌 키의 일부에 종속 = 부분 함수적 종속
